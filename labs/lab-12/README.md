# 12 - Каналы

## Задание
1) Реализовать программу, которая создает неименованный канал и порождает процесс. Родительский процесс закрывает дескриптор для чтения из канала и записывает в канал строку `Hi!`, затем ожидает завершения дочернего процесса. Дочерний процесс закрывает дескриптор для записи в канал, считывает строку из канала и выводит на экран.

2) Реализовать 2 программы: сервер и клиент. Сервер создает именованный канал и открывает его только на запись, записывает строку `Hi!` и завершается. Клиент открывает созданный канал сервером только на чтение, считывает строку из канала и выводит на экран. Удаляет канал.

3) Переписать командный интерпретатор из задания по процессам так, чтобы он поддерживал конструкции вида:
```bash
ls -la | grep a
```

## Структура проекта
```bash
├── bin              # Собранные бинарники
├── Makefile
├── README.md
└── src
    ├── 1.c          # Неименованный канал: parent -> child
    ├── server.c     # Именованный канал: сервер
    ├── client.c     # Именованный канал: клиент
    └── 3.c          # Мини-интерпретатор с поддержкой pipe
```

## Сборка и запуск
```bash
make                 # Сборка всех заданий
make run-1           # Запуск 1 задания
make run-server      # Запуск сервера
make run-client      # Запуск клиента
make run-3           # Запуск 3 задания
make clean           # Удаление бинарников
```

## Проверка
> 1 задание: родитель пишет в pipe, дочерний читает строку
```bash
make run-1
gcc -Wall -Wextra -Werror -std=c11 src/1.c -o bin/1
./bin/1
Hi!
```

> 2 задание: сервер пишет в FIFO, клиент читает и удаляет канал
```bash
make run-server
gcc -Wall -Wextra -Werror -std=c11 src/server.c -o bin/server
./bin/server

make run-client
gcc -Wall -Wextra -Werror -std=c11 src/client.c -o bin/client
./bin/client
Received message: Hi!
```

> 3 задание: интерпретатор поддерживает одну конструкцию с `|`
```bash
make run-3
```
